# Документ требований

## Введение

Личный калькулятор — это настольное приложение для выполнения базовых арифметических операций с интерфейсом, вдохновленным Windows Calculator. Калькулятор предоставляет пользователям простой и интуитивный способ выполнения математических вычислений.

## Глоссарий

- **Калькулятор**: Система, выполняющая математические вычисления
- **Дисплей**: Компонент интерфейса, отображающий текущее значение и результаты
- **Операнд**: Числовое значение, участвующее в вычислении
- **Оператор**: Математическая операция (+, -, ×, ÷)
- **Выражение**: Комбинация операндов и операторов для вычисления

## Требования

### Требование 1: Базовые арифметические операции

**Пользовательская история:** Как пользователь, я хочу выполнять базовые арифметические операции, чтобы производить математические вычисления.

#### Критерии приемки

1. КОГДА пользователь вводит два операнда и оператор сложения, Калькулятор ДОЛЖЕН вычислить и отобразить их сумму
2. КОГДА пользователь вводит два операнда и оператор вычитания, Калькулятор ДОЛЖЕН вычислить и отобразить их разность
3. КОГДА пользователь вводит два операнда и оператор умножения, Калькулятор ДОЛЖЕН вычислить и отобразить их произведение
4. КОГДА пользователь вводит два операнда и оператор деления, Калькулятор ДОЛЖЕН вычислить и отобразить их частное

### Требование 2: Обработка деления на ноль

**Пользовательская история:** Как пользователь, я хочу получать понятное сообщение при делении на ноль, чтобы понимать, что операция невозможна.

#### Критерии приемки

1. ЕСЛИ пользователь пытается разделить число на ноль, ТО Калькулятор ДОЛЖЕН отобразить сообщение "Деление на ноль невозможно"
2. КОГДА отображается сообщение об ошибке деления на ноль, Калькулятор ДОЛЖЕН очистить текущее выражение

### Требование 3: Ввод чисел

**Пользовательская история:** Как пользователь, я хочу вводить целые и десятичные числа, чтобы выполнять вычисления с различными типами чисел.

#### Критерии приемки

1. КОГДА пользователь нажимает цифровую кнопку, Калькулятор ДОЛЖЕН добавить цифру к текущему операнду на Дисплее
2. КОГДА пользователь нажимает кнопку десятичной точки, Калькулятор ДОЛЖЕН добавить десятичную точку к текущему операнду
3. ЕСЛИ текущий операнд уже содержит десятичную точку, ТО Калькулятор ДОЛЖЕН игнорировать повторное нажатие кнопки десятичной точки
4. Калькулятор ДОЛЖЕН поддерживать ввод чисел длиной до 16 цифр

### Требование 4: Очистка и сброс

**Пользовательская история:** Как пользователь, я хочу очищать текущий ввод или полностью сбрасывать калькулятор, чтобы начинать новые вычисления.

#### Критерии приемки

1. КОГДА пользователь нажимает кнопку "C" (Clear), Калькулятор ДОЛЖЕН очистить текущий операнд и отобразить "0"
2. КОГДА пользователь нажимает кнопку "CE" (Clear Entry), Калькулятор ДОЛЖЕН очистить только текущий операнд, сохраняя предыдущие операнды и оператор
3. КОГДА пользователь нажимает кнопку "C" дважды подряд, Калькулятор ДОЛЖЕН полностью сбросить все выражение

### Требование 5: Последовательные операции

**Пользовательская история:** Как пользователь, я хочу выполнять последовательные операции без нажатия кнопки равно, чтобы быстро производить цепочки вычислений.

#### Критерии приемки

1. КОГДА пользователь вводит операнд, затем оператор, затем второй операнд, затем другой оператор, Калькулятор ДОЛЖЕН вычислить результат первой операции перед применением второго оператора
2. КОГДА вычисляется промежуточный результат, Калькулятор ДОЛЖЕН отобразить его на Дисплее
3. Калькулятор ДОЛЖЕН сохранять точность вычислений до 10 знаков после запятой в промежуточных результатах

### Требование 6: Отображение результата

**Пользовательская история:** Как пользователь, я хочу видеть результат вычисления после нажатия кнопки равно, чтобы получить финальный ответ.

#### Критерии приемки

1. КОГДА пользователь нажимает кнопку "=" (равно), Калькулятор ДОЛЖЕН вычислить и отобразить результат текущего выражения
2. КОГДА результат отображается, Калькулятор ДОЛЖЕН округлить его до 10 знаков после запятой
3. КОГДА результат превышает 16 цифр, Калькулятор ДОЛЖЕН отобразить его в экспоненциальной нотации

### Требование 7: Операция смены знака

**Пользовательская история:** Как пользователь, я хочу изменять знак числа, чтобы работать с отрицательными значениями.

#### Критерии приемки

1. КОГДА пользователь нажимает кнопку "+/-", Калькулятор ДОЛЖЕН изменить знак текущего операнда на противоположный
2. КОГДА текущий операнд равен нулю, Калькулятор ДОЛЖЕН сохранить значение "0" при нажатии кнопки "+/-"

### Требование 8: Операция процента

**Пользовательская история:** Как пользователь, я хочу вычислять проценты, чтобы упростить процентные расчеты.

#### Критерии приемки

1. КОГДА пользователь вводит первый операнд, оператор сложения или вычитания, второй операнд и нажимает кнопку "%", Калькулятор ДОЛЖЕН вычислить процент от первого операнда
2. КОГДА пользователь вводит первый операнд, оператор умножения или деления, второй операнд и нажимает кнопку "%", Калькулятор ДОЛЖЕН разделить второй операнд на 100

### Требование 9: Удаление последней цифры

**Пользовательская история:** Как пользователь, я хочу удалять последнюю введенную цифру, чтобы исправлять ошибки ввода без полной очистки.

#### Критерии приемки

1. КОГДА пользователь нажимает кнопку "⌫" (Backspace), Калькулятор ДОЛЖЕН удалить последнюю цифру текущего операнда
2. ЕСЛИ текущий операнд содержит только одну цифру, ТО Калькулятор ДОЛЖЕН отобразить "0" после удаления

### Требование 10: Интерфейс пользователя

**Пользовательская история:** Как пользователь, я хочу видеть интерфейс в стиле Windows Calculator, чтобы иметь знакомый и удобный опыт использования.

#### Критерии приемки

1. Калькулятор ДОЛЖЕН отображать Дисплей в верхней части окна
2. Калькулятор ДОЛЖЕН отображать кнопки цифр от 0 до 9 в сеточной раскладке
3. Калькулятор ДОЛЖЕН отображать кнопки операторов (+, -, ×, ÷) справа от цифровых кнопок
4. Калькулятор ДОЛЖЕН отображать кнопки функций (C, CE, ⌫, %, +/-, =) в логически сгруппированных позициях
5. Калькулятор ДОЛЖЕН использовать цветовую схему, соответствующую стилю Windows Calculator

### Требование 11: Клавиатурный ввод

**Пользовательская история:** Как пользователь, я хочу использовать клавиатуру для ввода, чтобы быстрее выполнять вычисления.

#### Критерии приемки

1. КОГДА пользователь нажимает цифровую клавишу на клавиатуре, Калькулятор ДОЛЖЕН обработать её как нажатие соответствующей кнопки
2. КОГДА пользователь нажимает клавишу оператора (+, -, *, /), Калькулятор ДОЛЖЕН обработать её как нажатие соответствующей кнопки
3. КОГДА пользователь нажимает клавишу Enter, Калькулятор ДОЛЖЕН обработать её как нажатие кнопки "="
4. КОГДА пользователь нажимает клавишу Escape, Калькулятор ДОЛЖЕН обработать её как нажатие кнопки "C"
5. КОГДА пользователь нажимает клавишу Backspace, Калькулятор ДОЛЖЕН обработать её как нажатие кнопки "⌫"

### Требование 12: История вычислений

**Пользовательская история:** Как пользователь, я хочу видеть историю моих вычислений, чтобы отслеживать предыдущие операции.

#### Критерии приемки

1. КОГДА пользователь завершает вычисление нажатием "=", Калькулятор ДОЛЖЕН добавить выражение и результат в историю
2. Калькулятор ДОЛЖЕН отображать последние 20 записей истории
3. КОГДА пользователь нажимает на запись в истории, Калькулятор ДОЛЖЕН загрузить результат этой операции как текущий операнд
4. КОГДА пользователь очищает историю, Калькулятор ДОЛЖЕН удалить все записи из истории
